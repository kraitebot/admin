Design system

The component foundation for the Kraite operator console. Dark chrome, calm light content, terminal-green accent, monospaced numbers. Trading semantics are sacred — green is long / profit / safe, red is short / loss / risk, never inverted. Surfaces use a soft radius scale; numbers are the hero.

Foundations

One signal color — terminal green — on a neutral ink scale. A fixed P&L pair and a five-step Black Swan regime palette carry all market meaning. No decorative gradients.

Data table

The workhorse. Mono tabular figures, right-aligned numerics, sticky header, row hover, and a 600ms green/red cell flash when a value updates. P&L and percent columns read one step larger. Position tile

The hero working component. Direction governs the whole tile: green for longs, red for shorts — applied to the 2px border, the inverted ribbon header, the lifecycle track, the metric labels, and the accent values. The header band carries the market, side tag, leverage and ETA; the lifecycle track shows current price (PX) heading toward the take-profit target (TP) along a limit-order ladder; two metric rows below read the position's progress and prices. Every metric label has a ? affordance with a styled hover tooltip. The header also carries the coin glyph (CoinMarketCap icon with a mono-letter fallback), a four-dot timeframe-oscillation readout, and an actions (⋯) menu. Status badges sit beside the symbol: an opening tile is desaturated until the position is established, and a WAP'd tile (weighted-average-price re-entry) is flagged in amber.

Motion

Motion is purposeful and quick — it confirms a state change, never decorates. Two signature movements: a single highlight that slides to the selected target (sidebar rail, segmented control) and a content crossfade on route change. Both decelerate into place with an expo-out curve and no overshoot.

Projections

Revenue calendar

The Projections surface is a monthly calendar of daily revenue. Past days show realized P&L (strict green/red); future days show projected revenue, compounded forward from today's wallet at an observed daily rate. The current month is hybrid (realized → projected); past months are pure realized; future months are pure projection.

Hard rule — green/red are reserved for realized money. Projected cells carry the scenario tone (neutral = info-blue, pessimistic = red, optimistic = green) but desaturated, medium weight, and PROJ-tagged, so they're never confused with realized P&L. Today is the anchor — a neutral ink ring, never green. Amounts use compact notation ($1.20K · $3.45M · $1.2e26); two decimals on suffixed forms.

Projections

Scenario switch

A three-segment control (same family as the toolbar segmented control) that repaints all projected cells and totals. Each segment is self-documenting — it carries its anchor daily rate, so the operator sees the spread before clicking. Pessimistic = worst observed daily %, Neutral = midpoint of the observed range, Optimistic = best observed daily %.

The active pill takes the scenario tone (red / info-blue / green). Disabled + dimmed on past months (which carry no projection). Implementation note: the pill slides via a pure-percentage left transition — this engine won't interpolate a transition between calc() values, so width/inset are static and only left animates.

Totals strip & card--flat

A five-cell summary whose labels adapt to the month type. Current: started · made so far · projected (rest) · expected end · monthly %. The monthly-return value takes the scenario tone (not green) and a REAL · PROJ split sub so a positive figure is never misread as all-realized. Future months gain a cumulative-from-today band; past months swap to realized / ended at.
