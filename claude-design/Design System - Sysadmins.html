Sysadmin design system

The operator console (Kraite Admin - Sysadmins.html) is internal staff tooling — the platform's command surface over the whole fleet. It inherits the entire base Kraite design system (type, radius, spacing, motion, the card model, the form vocabulary). This document only covers what is specific to staff mode. For everything else, see the base system.

Principle 01
Violet = staff mode

A cool indigo accent replaces terminal-green for all chrome and brand, so operators never confuse this console with the green trader app.

Principle 02
Green is reserved for health

Because violet owns the brand, green / amber / red carry only semantic meaning — healthy, degraded, down — and P&L. Never decorative.

Principle 03
Operators act, and are logged

Destructive controls (kill switch, fleet config) are confirm-gated and attributed. The UI makes the blast radius explicit before you commit.

Foundations

Staff accent

The single override that defines staff mode. Three curated cool accents ship as a Tweak (default indigo); all share the same lightness and chroma, varying only hue. Text on the accent is white (--on-accent / --fg-on-accent flip from the base's black-on-green).

How it's applied

A small <style> block in the page head overrides the accent ramp on :root; the App then re-drives it live from the active Tweak. Everything that reads --accent (buttons, rail highlight, card borders, focus rings) follows automatically — no per-component change.

:root {
  --accent: #7b6cff;
  --accent-hover: #968aff;
  --accent-press: #5d4ee6;
  --accent-soft: #211c4a;
  --on-accent: #ffffff;      /* base is #001008 (dark) */
  --fg-on-accent: #ffffff;   /* rail active-item text */
  --border-focus: #7b6cff;
}

Chrome

Icon rail

112px fixed rail, dark chrome (#07090b), white snake mark. A single violet pill slides to the active item (420ms ease). Nine items vertically centered via an auto-margin wrapper that falls back to scroll on short viewports — never clipping. On ≤640px it becomes a bottom bar.

Engineering notes

⚠︎ Every button needs an explicit background

Tailwind Preflight is off, so a bare <button> renders the UA default (#C0C0C0 silver). Always set bg-transparent (or a real fill / inline style.background). The A_BTN_* presets already include one; plain buttons must add it.

Accent is the only theme change

Don't hard-code violet in components — read var(--accent) / use bg-accent / text-accent. The page-level override + the App's live setter handle the rest, which is what makes the accent Tweakable.

Green stays semantic

Never use green as a brand/CTA color here — it reads as "healthy / profit". Primary actions are violet (A_BTN_PRIMARY); positive states use the health/pnl tokens.

Shared-scope identifiers

Babel scripts share one global lexical scope — top-level const names must be unique across all loaded files (the A_/Ovw/Admin prefixes exist for this). Components are published via Object.assign(window, …).

Mock data → API

All datasets live in admin-kit.jsx / pages-data.jsx as plain arrays — the shape each section's API should return. Swap them for fetches without touching the view components.
